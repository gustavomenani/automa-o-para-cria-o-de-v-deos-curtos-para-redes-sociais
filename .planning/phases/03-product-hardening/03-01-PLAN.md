<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser->login action | Untrusted credentials enter server action. |
| server->cookie | Signed session leaves server into browser storage. |
| server->database | Password hash and user ownership fields are persisted. |

| T-03-01-02 | I | `src/features/auth/actions.ts` | mitigate | Use generic invalid-login feedback and never reveal whether an email exists. |

</threat_model>
