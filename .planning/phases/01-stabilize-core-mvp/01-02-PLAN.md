<objective>
Make generation outcomes understandable to the user and correct stale documentation.

Purpose: After caption stabilization, the MVP should not surprise the user. Failed generation, uncertain caption sync, Gemini/Manus limitations, and manual-upload fallback must be visible without exposing raw subprocess/provider details.

Output: Safer user-facing feedback in existing content flows plus README corrections for Manus and Whisper behavior.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|---|---|
| service/integration errors -> redirect query params | Internal errors can become visible in browser URLs and UI banners. |
| persisted `ContentProject.errorMessage` -> review page | Stored generation warning/error text is rendered to the user. |
| README operational docs -> developer setup | Documentation can cause unsafe secret handling or incorrect assumptions about real integrations. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|---|---|---|---|---|
| T-01-02-01 | Information Disclosure | `src/features/content/actions.ts` and `src/app/contents/[id]/page.tsx` | mitigate | Add or reuse safe-message mapping so FFmpeg stderr, local paths, provider payloads, stack traces, and secret-looking values are not placed in query strings or rendered from persisted `ContentProject.errorMessage`. |
| T-01-02-02 | Tampering | review page query params | mitigate | Keep query params as display-only feedback; do not derive project state from them. Decode only for text display in `FeedbackBanner`. |
| T-01-02-03 | Repudiation | generation warnings | accept for Phase 1 | Store concise user-facing warning text in `ContentProject.errorMessage`. Full structured job logs are deferred to worker/ops phases. |
| T-01-02-04 | Information Disclosure | README secret guidance | mitigate | State that Gemini and Manus keys belong in `.env` or `/settings`, never in client code, screenshots, or committed files. |
</threat_model>
