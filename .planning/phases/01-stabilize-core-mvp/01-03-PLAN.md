<objective>
Create and run an end-to-end local smoke validation plan for Phase 1.

Purpose: Caption fixes and UI warnings only matter if the whole brownfield MVP still works: create content, upload or generate assets, render MP4, review, download, schedule, and delete without surprising the user.

Output: A reusable smoke checklist plus dated execution evidence that records pass/fail/blocked results and follow-up notes. This plan may edit only the checklist file.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|---|---|
| local browser -> server actions/API routes | User form data and uploaded files enter server-side mutation paths. |
| local filesystem storage -> public file route | Files under `storageRoot` are exposed through `/api/files/[...path]`. |
| generated MP4/download -> user machine | Binary output is downloaded from local storage. |
| schedule form -> database | Platform, date, time, and caption create `ScheduledPost` rows without real social publishing. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|---|---|---|---|---|
| T-01-03-01 | Tampering | upload validation smoke step | mitigate | Include explicit invalid-file smoke step proving unsupported MIME types are rejected before storage. |
| T-01-03-02 | Information Disclosure | file download smoke step | mitigate | Include traversal/path-guessing check against `/api/files/../` style paths; expected result is rejection or not found, never arbitrary file disclosure. |
| T-01-03-03 | Denial of Service | local FFmpeg/Whisper smoke run | accept for Phase 1 | Use one small local fixture audio and 2-4 images. Upload size/duration limits and worker isolation are deferred. |
| T-01-03-04 | Spoofing | no-auth local MVP | accept for Phase 1 | Document that Phase 1 is local-only and unauthenticated. Auth and ownership are Phase 3 and must not be implemented here. |
| T-01-03-05 | Repudiation | smoke evidence | mitigate | Require checklist results to be recorded in the plan summary with exact commands, URLs, and pass/fail notes. |
</threat_model>
