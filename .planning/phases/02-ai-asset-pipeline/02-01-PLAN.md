<objective>
Create the persistence and contract foundation for Manus/Gemini run tracking.

Purpose: Phase 2 requires provider task/status/log persistence before orchestration and UI updates. Without this foundation, later plans cannot reliably explain partial AI outputs.

Output: Prisma schema + migration for asset generation runs, typed service helpers for run lifecycle, and content relation types ready for wave 2 and wave 3.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|---|---|
| external provider metadata -> database | Untrusted status/error text from Manus/Gemini is persisted for UI visibility. |
| server service -> UI consumers | Stored summaries can later be rendered in `/contents/[id]`. |
| schema changes -> runtime app | Prisma model drift can break writes silently if schema/client are not synchronized. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|---|---|---|---|---|
| T-02-01-01 | Tampering | provider summary persistence | mitigate | Persist normalized summary fields and status enums only; do not store arbitrary raw provider payload blobs in display fields. |
| T-02-01-02 | Information Disclosure | run summary text | mitigate | Add redaction helper in run service to strip API-key-like patterns, filesystem paths, and stack traces before persistence. |
| T-02-01-03 | Repudiation | generation lifecycle transitions | mitigate | Store `startedAt`, `finishedAt`, `status`, `providerTaskId`, and append-only summary updates to preserve traceability of each attempt. |
| T-02-01-04 | Denial of Service | schema mismatch between code and DB | mitigate | Add a mandatory schema sync task (`prisma generate` + `prisma db push`) before wave 2 code is implemented. |
</threat_model>
