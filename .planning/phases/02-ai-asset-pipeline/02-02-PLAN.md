<objective>
Implement Manus-first prompt-to-assets orchestration with resilient fallback behavior.

Purpose: Phase 2 requires a reliable automated path from prompt to usable assets while preserving manual upload fallback and avoiding secret leakage.

Output: A new orchestrator service, expanded Manus normalization, action wiring for provider priority, and robust handling for partial/failed provider responses.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|---|---|
| external provider APIs -> orchestration service | Untrusted payloads from Manus/Gemini are normalized and persisted. |
| orchestration errors -> redirect query params | Internal failures can leak implementation details if not sanitized. |
| provider credentials -> runtime logs/persistence | API keys may leak through error strings if not redacted. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|---|---|---|---|---|
| T-02-02-01 | Information Disclosure | provider error propagation | mitigate | Normalize provider failures to stable Portuguese messages and redact key/path/stack-trace patterns before redirect/persistence. |
| T-02-02-02 | Tampering | Manus attachment parsing | mitigate | Accept only expected attachment types and URLs; map missing/invalid attachments to partial status instead of trusting payload shape blindly. |
| T-02-02-03 | Denial of Service | in-request provider polling/generation | accept for this phase | Keep current in-request execution for MVP, but persist status transitions and explicit failure reasons to support later queue migration. |
| T-02-02-04 | Repudiation | provider status transitions | mitigate | Persist transitions (`QUEUED` -> `RUNNING` -> terminal state) with summary events per run id to preserve audit trail. |
| T-02-02-05 | Elevation of Privilege | server-only provider clients | mitigate | Keep API keys and provider calls inside server modules (`actions/services/integrations`) and never expose secret values to client components. |
</threat_model>

- Export `runAssetPipelineForProject({ projectId, prompt })` that:
  1. starts a run via `startAssetGenerationRun` with provider `MANUS` when Manus key/settings exist

  3. falls back to Gemini when Manus is unavailable, fails, or returns partial assets insufficient to auto-generate video
  4. persists run transitions and summary/missing-assets metadata using the run service
- Reuse existing Gemini capabilities (`generateReelsPlan`, `generateSceneImages`, `generateNarrationAudio`) but move provider-selection decisions into orchestrator.
- Persist project title/caption updates and created media rows in one transaction where possible, preserving current `saveGeneratedAsset` behavior.
- Define normalized orchestrator return shape with:
  - `providerUsed`
  - `plan`
  - `images`
  - `audio`
  - `status`
  - `missingAssets`
  - `warnings`
  - `runId`
- Ensure warnings classify at least: quota/rate-limit, invalid-key/permission, model-unavailable/high-demand, empty-attachment/no-media, and manual-action-required.
- Never include raw provider payloads, key values, or stack traces in returned warning strings.
