## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser to Server Action | Authenticated form/API input requests background work. |
| browser to job status API | User asks for job state and retry behavior. |
| UI to logs | Persisted operational logs are displayed to the user. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-03-01 | Spoofing | retry/status endpoints | mitigate | Use `requireUser` and owner-scoped JobRun queries. |
| T-04-03-02 | Tampering | retry action | mitigate | Allow retry only for failed owned jobs and use typed persisted payloads. |
| T-04-03-03 | Information Disclosure | JobStatusPanel | mitigate | Display only safe summaries/logs created by Plan 04-01 redaction. |
| T-04-03-04 | Denial of Service | enqueue actions | mitigate | Use deterministic job ids for duplicate-prone jobs and preserve upload/generation guards. |
</threat_model>
