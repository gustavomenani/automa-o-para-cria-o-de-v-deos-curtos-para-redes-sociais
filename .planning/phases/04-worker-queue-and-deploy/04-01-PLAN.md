<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| web action/API to queue | Authenticated user input becomes asynchronous worker payload. |
| worker/provider/process to database | Provider, FFmpeg, Python, and Redis failures become persisted diagnostics. |
| Redis to Postgres | Redis queue state is not trusted as the only source of user-visible status. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01-01 | Spoofing | `enqueueJob` payloads | mitigate | Require `userId` in typed payloads and have later entry points derive it from `requireUser`, never request body. |
| T-04-01-02 | Tampering | job payload/status | mitigate | Persist JobRun before queue add and use service-only status transition helpers. |
| T-04-01-03 | Information Disclosure | JobRun logs | mitigate | Redact secrets, local paths, raw payloads, and stack traces before persistence. |
| T-04-01-04 | Denial of Service | retry settings | mitigate | Add bounded attempts, exponential backoff, and deterministic job ids for duplicate-prone jobs. |
</threat_model>
